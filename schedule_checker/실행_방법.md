# AutoFlow 앱 실행 방법

브라우저에서 개발 현황 화면을 보려면 아래 방법 중 하나를 쓰면 됩니다.

**처음 사용하시나요?** 로그인·업무를 쓰려면 **한 번만** 설정이 필요합니다.  
→ **[docs/사용전_설정_초보자용.md](docs/사용전_설정_초보자용.md)** 에 단계별로 자세히 적어 두었으니, 그대로 따라 하시면 됩니다.

**코드 수정 후 서버 재시작이 필요할 때**  
→ 재시작은 **AI가 터미널에서 대신** 수행합니다(4000 포트 정리 후 서버 재실행).  
→ 사용자는 **http://localhost:4000** 을 **새로고침(F5)** 만 하면 됩니다.

---

## 방법 0: 더블클릭으로 실행 (가장 쉬움)

1. 프로젝트 폴더(`schedule_checker`)를 연다.
2. **"AutoFlow ON.bat"** 파일을 **더블클릭**한다.
3. 서버 창이 뜨고, 약 10초 후 브라우저가 자동으로 http://localhost:4000 을 연다.
4. 앱을 끌 때는 **서버 창(검은 창)** 을 닫으면 된다.

**아이콘 바꾸기**: `AutoFlow ON.bat` 에 마우스 오른쪽 클릭 → **바로 가기 만들기** → 만들어진 바로 가기에서 오른쪽 클릭 → **속성** → **아이콘 변경**에서 원하는 아이콘을 고를 수 있다.

---

## 방법 1: F5 키 (Cursor에서)

1. Cursor에서 이 프로젝트가 열린 상태에서 **키보드 F5** 를 누릅니다.
2. 처음이면 **"구성 선택"** 이 나올 수 있습니다. 목록에서 **"AutoFlow 실행 (개발 서버 + 브라우저)"** 를 고르고 확인합니다.
3. 잠시 후 터미널에 서버가 뜨고, **브라우저가 자동으로** http://localhost:4000 이 열립니다.

---

## 방법 2: 명령 팔레트

1. **Ctrl+Shift+P** (Mac: **Cmd+Shift+P**) 를 눌러 명령 팔레트를 엽니다.
2. **"디버깅"** 또는 **"Debug"** 를 입력합니다.
3. **"디버그: 디버깅 시작"** 또는 **"Debug: Select and Start Debugging"** 를 선택합니다.
4. 목록에서 **"AutoFlow 실행 (개발 서버 + 브라우저)"** 를 선택합니다.
5. 서버가 켜지면 브라우저가 자동으로 http://localhost:4000 을 엽니다.

---

## 방법 3: 터미널만 사용

1. Cursor 하단 **터미널** 탭을 엽니다.
2. 다음 중 하나를 입력하고 Enter 합니다.
   ```bash
   npm run dev
   ```
   **"스크립트를 실행할 수 없습니다" / ExecutionPolicy 오류가 나면** 아래 중 하나를 사용하세요.
   ```bash
   npm.cmd run dev
   ```
   또는 (실행 정책을 한 번만 바꾸려면):
   ```powershell
   Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned
   ```
   입력 후 다시 `npm run dev` 를 실행합니다.
3. **"Ready in ..."** 이 보이면 브라우저에서 **http://localhost:4000** 주소를 직접 엽니다.

---

## Run and Debug 아이콘 위치 (방법 1·2가 안 될 때)

- Cursor **맨 왼쪽 세로 줄**에 작은 아이콘들이 있습니다.
- 그중 **벌레 모양 + 재생(▶)** 아이콘 = **Run and Debug** 입니다.
- 클릭하면 왼쪽 패널에 **실행/디버그** 화면이 열리고, 위쪽 **드롭다운**에서 **"AutoFlow 실행 (개발 서버 + 브라우저)"** 를 고른 뒤 **녹색 재생 버튼(▶)** 을 누르면 됩니다.

---

## ⚠️ "EADDRINUSE: address already in use :::4000" 가 나올 때

**의미**: 4000번 포트를 **이미 다른 프로그램이 쓰고 있어서** 새 서버가 켜지지 않는 상태입니다.  
보통 **예전에 켜 둔 개발 서버**(다른 터미널 창, 또는 AutoFlow ON.bat으로 연 검은 창)가 아직 떠 있을 때 발생합니다.

**해결 방법 (둘 중 하나)**:

### 1) 이미 떠 있는 서버 창부터 끄기 (가장 간단)

- **AutoFlow ON.bat**으로 실행했다면: **제목이 "AutoFlow"인 검은 명령 창**을 찾아서 **X로 닫기**.
- **Cursor 터미널**에서 `npm run dev` 를 켰다면: 그 **터미널 탭**에서 **Ctrl+C** 한 번 누르기.
- 그 다음에 다시 **AutoFlow ON.bat** 더블클릭 또는 `npm run dev` 실행.

### 2) 포트 4000을 쓰는 프로그램을 직접 종료하기

1. **명령 프롬프트(cmd)** 또는 **PowerShell**을 연다 (Cursor 터미널도 가능).
2. 아래를 **한 줄씩** 입력하고 Enter 한다.
   ```bash
   netstat -ano | findstr :4000
   ```
3. 맨 오른쪽에 **숫자(PID)** 가 보인다. 예: `12345`.
4. 그 숫자를 넣어서 다음을 실행한다 (예: PID가 12345일 때).
   ```bash
   taskkill /PID 12345 /F
   ```
5. 다시 **AutoFlow ON.bat** 또는 `npm run dev` 로 서버를 켠다.

**한 번에 하기**: 프로젝트 폴더에 **"AutoFlow 재시작.bat"** 이 있으면, 이걸 더블클릭해서 **4000번 포트를 비운 뒤** 서버를 자동으로 켤 수 있다.

---

## ⚠️ "스크립트를 실행할 수 없습니다" / ExecutionPolicy 오류가 날 때

PowerShell에서 `npm run dev` 시 위와 같은 오류가 나면:

1. **우회 실행** (가장 간단): 터미널에 아래를 입력합니다.
   ```bash
   npm.cmd run dev
   ```
2. **실행 정책 변경** (한 번 설정하면 앞으로 `npm` 그대로 사용 가능): PowerShell에서 한 번만 실행합니다.
   ```powershell
   Set-ExecutionPolicy -Scope CurrentUser -ExecutionPolicy RemoteSigned
   ```
   이후에는 `npm run dev` 를 그대로 사용할 수 있습니다.

F5로 실행할 때는 **npm.cmd** 를 사용하도록 설정해 두었습니다. 터미널에서만 위 우회 방법을 쓰면 됩니다.

## ⚠️ npm install 시 "TAR_ENTRY_ERROR" 가 반복될 때

**원인**: 프로젝트가 **Google Drive(내 드라이브)** 같은 **동기화 폴더** 안에 있으면, npm이 수천 개의 작은 파일을 쓸 때 동기화 프로그램이 파일을 잠그거나 간섭해서 이 오류가 납니다.

**해결 방법 (둘 중 하나 권장)**:

### 1) 프로젝트를 로컬 폴더로 복사해서 설치 (권장)

1. **Google Drive 동기화 폴더 밖**에 폴더 하나 만든다. 예: `C:\dev` 또는 `D:\dev`
2. `schedule_checker` **전체 폴더**를 그 안으로 **복사**한다. (예: `C:\dev\schedule_checker`)
3. **복사한 폴더**에서만 아래를 실행한다.
   - `AutoFlow ON.bat` 더블클릭  
   - 또는 터미널에서 `npm.cmd install` 후 `npm.cmd run dev`
4. 앞으로는 **복사한 폴더**를 Cursor로 열고, 여기서 개발·실행하면 된다.  
   (원본은 드라이브에 그대로 두고 백업/동기화용으로만 써도 됨)

### 2) 지금 위치에서 할 때 (드라이브 일시 중지)

1. **Google Drive 앱**에서 **동기화를 일시 중지**한다.
2. 프로젝트 폴더에서 터미널을 열고 `npm.cmd install` 실행한다.
3. 설치가 끝나면 다시 동기화를 켜도 된다.  
   (단, 이후에도 `node_modules` 때문에 동기화가 느려지거나 오류가 날 수 있어서, 가능하면 **1)처럼 로컬 폴더로 옮기는 것을 추천**한다.)

설치가 끝나면 `node_modules` 폴더가 생기고, 그 다음부터는 **AutoFlow ON.bat** 더블클릭으로 서버 실행이 가능합니다.

## Google Drive에서 작업해도 되나요?

- **코드 편집·저장**: Google Drive 폴더에서 해도 됩니다. 소스 파일(.tsx, .json 등) 편집은 동기화와 충돌이 거의 없습니다.
- **실행·설치**: **`npm install`** 과 **`npm run dev`**(또는 AutoFlow ON.bat)는 **Drive 밖 로컬 폴더**(예: `C:\dev\schedule_checker`)에서 하는 것을 권장합니다. Drive 안에서 설치/실행하면 TAR_ENTRY_ERROR나 `next`를 찾을 수 없는 오류가 다시 날 수 있습니다.
- **권장 흐름**: Drive에 원본을 두고 **편집** → 필요할 때 로컬 복사본에 **동기화 또는 수동 복사** → 로컬에서 **설치·실행** 후 브라우저로 확인.

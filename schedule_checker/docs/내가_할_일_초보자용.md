# AutoFlow 사용 전에 내가 할 일 (초보자용)

앱을 **처음 켜서 쓰기 전**에, 아래 순서대로 **한 번만** 하시면 됩니다.

---

## 1단계: 프로젝트 폴더 준비

- 이 프로젝트를 **구글 드라이브 동기화 폴더 말고**, 그냥 **내 PC 안의 폴더**에 두는 것을 권장합니다.  
  (예: `C:\dev\schedule_checker` 또는 `S:\Cursor\schedule_checker`)
- 그래야 `npm install` 할 때 오류가 잘 안 난다고 보시면 됩니다.

---

## 2단계: Node.js 설치 확인

1. **명령 프롬프트** 또는 **PowerShell**을 연다.
2. 아래를 입력하고 Enter를 누른다.
   ```bash
   node -v
   ```
3. `v18.xxx` 또는 `v20.xxx` 처럼 **버전 번호가 나오면** OK.  
   “인식할 수 없습니다”라고 나오면, [Node.js 공식 사이트](https://nodejs.org/)에서 LTS 버전을 설치한 뒤 다시 2번을 확인한다.

---

## 3단계: 의존성 설치 (npm install)

1. **명령 프롬프트**에서 이 프로젝트 **폴더 안**으로 들어간다.  
   예: `cd S:\Cursor\schedule_checker`
2. 아래를 입력하고 Enter를 누른다.
   ```bash
   npm install
   ```
3. 끝날 때까지 기다린다.  
   “TAR_ENTRY_ERROR” 같은 오류가 나오면, 1단계처럼 **동기화 폴더 밖**으로 프로젝트를 옮긴 뒤 같은 폴더에서 다시 `npm install`을 한다.

---

## 4단계: Firebase 설정 (한 번만)

### 4-1. Firebase 콘솔에서 할 일

1. 브라우저에서 [Firebase 콘솔](https://console.firebase.google.com/)에 들어간다.
2. 사용할 **프로젝트**를 선택한다 (또는 새로 만든다).
3. **Authentication(인증)** 메뉴로 들어가서:
   - **로그인 방법** 탭에서 **이메일/비밀번호**를 “사용”으로 켠다.
   - **Google** 로그인도 쓰려면 “사용”으로 켠다.
4. **프로젝트 설정**(톱니바퀴) → **일반** → **내 앱**에서 **웹 앱**을 추가하거나, 이미 만든 웹 앱을 선택한다.
5. **SDK 설정**에서 나오는 값들을 복사해 둔다.  
   (`apiKey`, `authDomain`, `projectId`, `storageBucket`, `messagingSenderId`, `appId`)

### 4-2. 프로젝트 안에 환경 변수 파일 만들기

1. 이 프로젝트 **폴더 안**에 `.env.local` 이라는 파일을 만든다.  
   (이미 있으면 그걸 연다.)
2. 아래처럼 적어 넣는다. **`여기에_넣기`** 자리에는 4-1에서 복사한 값을 그대로 붙여 넣는다.

   ```env
   NEXT_PUBLIC_FIREBASE_API_KEY=여기에_넣기
   NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN=여기에_넣기.firebaseapp.com
   NEXT_PUBLIC_FIREBASE_PROJECT_ID=여기에_넣기
   NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET=여기에_넣기.firebasestorage.app
   NEXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID=여기에_넣기
   NEXT_PUBLIC_FIREBASE_APP_ID=여기에_넣기
   ```

3. 저장한 뒤, **개발 서버를 한 번 종료했다가 다시 켜야** 이 값이 반영된다.

---

## 5단계: Firestore 규칙·인덱스 배포 (한 번만)

1. **명령 프롬프트**에서 이 프로젝트 **폴더**로 이동한 뒤, 아래를 입력한다.
   ```bash
   npx firebase-tools login
   ```
2. 브라우저가 열리면 **Google 계정**으로 로그인하고, 터미널에 “Success”가 나오면 다음으로.
3. 같은 터미널에서 아래를 입력한다.
   ```bash
   npm run firestore:deploy
   ```
4. 프로젝트를 묻는 말이 나오면, 쓰는 **Firebase 프로젝트**를 선택한다.
5. “Deploy complete!”가 나오면 끝이다.  
   “인덱스가 필요합니다”라는 오류가 나오면, 오류 메시지에 나온 **링크**를 열어서 인덱스를 만들어 준다.

---

## 6단계: 앱 실행

- **방법 A**: 프로젝트 폴더에 있는 **`AutoFlow ON.bat`** 파일을 **더블클릭**한다.  
  → 서버가 켜지고, 잠시 후 브라우저가 `http://localhost:4000` 으로 열린다.
- **방법 B**: Cursor(또는 VS Code)에서 이 프로젝트를 연 상태에서 **F5**를 누르거나, **실행 및 디버그**에서 “AutoFlow 실행”을 선택해 실행한다.

실행 후 브라우저에서 **회원가입/로그인**을 하면 업무 목록 화면을 쓸 수 있다.

---

## 7단계: 로그인이 새로고침 후에도 유지되게

- 앱에서는 **브라우저(IndexedDB·localStorage)** 에 로그인 상태를 저장해 두도록 되어 있다.  
  그래서 **새로고침**해도 다시 로그인하지 않아도 된다.
- 만약 **새로고침할 때마다 로그인 화면**으로 돌아간다면:
  1. **시크릿/프라이빗 창**이 아니라 **일반 창**에서 연다.
  2. **사이트 데이터를 한 번 비운 뒤 다시 로그인**해 본다.  
     - Chrome: 주소창 왼쪽 자물쇠 클릭 → "이 사이트의 쿠키 및 사이트 데이터" → **데이터 삭제**  
     - 또는 F12 → **Application** 탭 → **Storage** → **Clear site data**
  3. 페이지를 **새로 고침**한 뒤, **다시 로그인**한다. (이후 새로고침 시 유지되는지 확인)
  4. 위 4단계의 `.env.local` 이 **제대로 저장**되어 있고, **서버를 한 번 재시작**한 뒤에 다시 접속해 본다.

# 매일 9시 업무 보고서 이메일 (인포그래픽 A4) 방안

매일 **오전 9시**에 **인포그래픽 디자인이 적용된 A4 사이즈 업무 보고서**를 **이메일**로 받을 수 있게 하는 방안입니다.

---

## 1. 요구사항 정리

| 항목 | 내용 |
|------|------|
| **발송 시각** | 매일 오전 9시 (한국 시간 KST 권장, 또는 사용자별 시간대 설정) |
| **형태** | 이메일 (HTML 본문 또는 PDF 첨부) |
| **디자인** | 인포그래픽 스타일, **A4 사이즈** (210×297mm)에 맞는 레이아웃 |
| **내용** | 전날 완료, 오늘 예정, 이번 주 마감, 진행 중 목표 등 요약 |

---

## 2. 보고서에 넣을 내용 (예시)

- **헤더**: 로고/앱명, 보고 일자(예: 2025년 3월 15일 업무 보고서).
- **한 줄 요약**: "어제 3건 완료 · 오늘 5건 예정".
- **어제 완료**: 완료한 할 일 목록 (제목, 완료 시각).
- **오늘 할 일**: 마감일이 오늘인 것 + 선택적으로 "진행 중인데 마감일 없음" n건.
- **이번 주 마감**: 요일별 또는 리스트로 이번 주 마감 할 일.
- **목표 진행률**(목표–하위 업무 도입 시): 목표별 진행률(예: 2/5 완료).
- **푸터**: "AutoFlow에서 자세히 보기" 링크, 수신 설정 변경 링크.

---

## 3. 인포그래픽·A4 디자인 방향

- **A4 비율**: 210mm × 297mm → 웹/이메일에서는 **794×1123px**(96dpi) 또는 **2480×3508px**(300dpi 인쇄용) 등으로 가이드. 이메일 본문은 **폭 600px 전후**로 두고, **PDF 첨부**를 A4로 만들면 인쇄·보관에 유리.
- **인포그래픽 요소**  
  - 숫자 강조: "3건 완료", "5건 오늘" 등을 큰 폰트·아이콘으로.  
  - 섹션별 구분: 카드형 박스, 구분선, 작은 아이콘(체크, 캘린더, 깃발 등).  
  - 색: 앱 메인 컬러(인디고) + 완료(초록), 예정(주황/노랑), 주의(빨강) 등 일관된 톤.  
  - 차트(선택): 진행률 막대, 요일별 건수 등 간단한 시각화.
- **이메일 호환**  
  - HTML 이메일: 테이블 기반 레이아웃, 인라인 CSS 위주. 많은 클라이언트에서 동일하게 보이도록.  
  - A4 느낌을 주려면: 본문 폭을 고정(예: 600px), 세로로 길게 구성. 실제 A4 출력은 **PDF 첨부**로 제공하는 편이 안정적.

## 4. 기술 구현 방안

### 4.1 발송 주기 (스케줄)

- **Vercel Cron**: `vercel.json`에 매일 9시 KST 실행 설정.  
  - 예: `0 0 * * *` (UTC 0시) = KST 9시. (또는 `0 9 * * *`는 Vercel이 UTC 기준이므로 KST 9시가 되도록 cron 식 조정.)
- **실제 발송**: Cron이 호출하는 **API 라우트**에서 "오늘 발송 대상 사용자"를 구한 뒤, 사용자별로 보고서 생성 + 이메일 발송.

### 4.2 발송 대상 사용자

- **옵션 1**: Firestore에 `users` 또는 `user_settings` 컬렉션.  
  - 문서 ID = `uid`.  
  - 필드: `email`, `dailyReportEnabled: true`, `reportTimeZone: "Asia/Seoul"` 등.  
  - 가입 시 또는 "설정"에서 "매일 업무 보고서 받기" on 시 문서 생성/갱신.
- **옵션 2**: Firebase Auth에서 이메일 가져오기 + "기본적으로 모두 발송" 후, 설정에서 끄기만 할 수 있게.
- **수집**: Cron API에서 `dailyReportEnabled === true`인 사용자 목록 조회, 각자 `email`로 발송.

### 4.3 보고서 생성

- **데이터**: 기존 Firestore `tasks` (및 추후 `parentTaskId` 등)를 Admin SDK로 조회.  
  - 어제 완료: `completed === true`이고 `updatedAt`이 어제 구간.  
  - 오늘 할 일: `dueDate`가 오늘, 미완료.  
  - 이번 주: `dueDate`가 이번 주, 미완료.
- **HTML 템플릿**:  
  - Node에서 **HTML 문자열** 또는 **React 컴포넌트를 HTML로 렌더(예: @react-email/components 또는 문자열 치환)**.  
  - 인포그래픽 레이아웃: 상단 요약 → 섹션별 블록 → 푸터. A4 비율에 맞게 한 페이지 분량으로 제한.
- **PDF 생성(선택)**:  
  - **Puppeteer** 또는 **@react-pdf/renderer** 등으로 HTML → PDF.  
  - 페이지 크기 A4(210mm×297mm) 지정.  
  - 이메일에 **PDF 첨부**로 붙이거나, 링크로 다운로드 제공.

### 4.4 이메일 발송

- **서비스**: **Resend**, **SendGrid**, **AWS SES**, **Mailgun** 등 중 하나 사용.  
  - Resend/SendGrid는 Next.js API에서 바로 호출 가능, 발신 도메인 인증 필요.
- **발신**:  
  - From: `AutoFlow <noreply@yourdomain.com>` 등.  
  - To: 사용자 `email`.  
  - Subject: `[AutoFlow] 2025년 3월 15일 업무 보고서`.  
  - 본문: HTML. 첨부: PDF 파일(선택).
- **환경 변수**: `EMAIL_API_KEY`, `EMAIL_FROM` 등.

### 4.5 API 라우트 예시

- **`/api/cron/daily-report`** (또는 기존 `/api/automation/daily-summary` 확장).  
  - Method: GET.  
  - 헤더에 **Cron Secret** 검증 (Vercel Cron만 호출하도록).  
  - 처리 순서:  
    1) 보고서 수신 동의 사용자 목록 조회.  
    2) 각 사용자에 대해:  
       - Firestore에서 해당 사용자 할 일 데이터 조회.  
       - HTML(및 필요 시 PDF) 생성.  
       - 이메일 발송.  
    3) 로그/에러 처리 (실패 시 재시도 또는 다음날 재발송 정책은 선택).

---

## 5. 설정 화면 (사용자)

- **설정 페이지** 또는 **업무 페이지 내 설정**:  
  - "매일 오전 9시 업무 보고서 이메일 받기" **ON/OFF**.  
  - ON 시 Firestore `user_settings` 등에 `dailyReportEnabled: true`, `email` 저장.  
  - (선택) 시간대 또는 받는 시간 변경.

---

## 6. 구현 순서 제안

1. **Firestore**: `user_settings`(또는 기존 사용자 문서)에 `dailyReportEnabled`, `email` 필드.  
2. **이메일 서비스 가입**: Resend 등에서 API 키 발급, 발신 도메인 인증.  
3. **HTML 템플릿**: 인포그래픽 스타일 A4 비율 한 페이지 분량으로 작성.  
4. **보고서 데이터 로직**: 어제 완료/오늘/이번 주 조회.  
5. **API 라우트**: Cron → 사용자 조회 → 보고서 생성 → 이메일 발송.  
6. **Vercel Cron**: 매일 9시 KST에 해당 API 호출하도록 설정.  
7. **설정 UI**: 보고서 수신 ON/OFF (및 이메일 주소 표시).

PDF 첨부는 1~6이 안정된 뒤에 추가해도 됩니다. 먼저 **HTML 이메일만**으로 인포그래픽 A4 느낌의 보고서를 보내고, 필요 시 PDF를 붙이는 순서를 권장합니다.

원하시면 **HTML 템플릿 예시**(인포그래픽 블록 구성)나 **Cron + API 라우트 코드 골격**부터 구체적으로 작성해 드리겠습니다.

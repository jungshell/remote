# AutoFlow 앱 비판적 검토 및 개선·제도 제안

전체 앱을 비판적 시각으로 검토하고, **구현 방향**과 **업무 효율성·제도·시스템** 관점에서 개선·발전 방안을 제안합니다.

---

## 1. 현재 앱 구현 방향 요약

| 영역 | 현재 상태 | 강점 | 약점 |
|------|-----------|------|------|
| **할 일 관리** | CRUD, 목표·하위 업무, 목록/주간/월간 뷰 | 구조가 분명함, Firestore 연동 | 입력이 다단계, 일시 선택이 번거로움 |
| **일시 입력** | `datetime-local` 단일 입력 + 기본값(오늘 9시/내일 9시) | 정확한 날짜·시간 지정 가능 | 빠른 선택 옵션 부족 → **퀵 프리셋 도입으로 개선** |
| **세부 업무** | 목표(isGoal) 아래에만 하위 업무 추가 가능 | 목표 단위 진행률 표시 | 일반 할 일에는 세부 업무 UI 없음 → **일반 할 일에도 펼치기·세부 추가 가능하도록 개선** |
| **보고·설정** | 매일 9시 이메일 보고서 on/off | 자동 요약 수신 | Cron·이메일 실제 동작 여부, 시간대·형식 사용자화 부족 |
| **네비** | 업무 / 연락처·알림·템플릿(준비 중) | 단순 | 메인(업무)이 시각적으로 덜 강조됨 |

## 2. 비판적 검토 (구현·UX)

### 2.1 진입 장벽
- **문제**: "할 일 하나만 빨리 넣고 싶다"는 사용자에게 보기/필터/대시가 먼저 눈에 들어와 클릭·스크롤이 많음.
- **개선**: 상단에 **한 줄 입력 + Enter 또는 [추가]**로 즉시 추가(마감일은 기본값 또는 추가 후 수정). 상세(마감·우선순위·장소)는 "상세 옵션" 접기로 두기. (자세한 UX는 `docs/사용편의성_개선방향_비판.md` 참고.)

### 2.2 일시 입력
- **개선 완료**: 기한(일시)에 **퀵 프리셋**(오늘 18:00, 내일 9:00, 내일 18:00, 모레 9:00, 금요일 18:00, 다음주 월 9:00) 버튼을 두어 한 번에 선택 가능. 필요 시 아래 `datetime-local`로 세부 조정.

### 2.3 세부 업무(파생 업무) 입력·관리 방법

**현재 방식(및 개선 후):**
1. **목표 단위**: 할 일 추가 시 **"목표로 추가"**를 체크하면 최상위 목표가 됨. 목록에서 해당 항목 옆 **▼**를 누르면 **하위 업무 목록**과 **"하위 업무 추가"** 입력창이 나타남. 여기서 입력한 항목은 자동으로 해당 목표의 세부 업무로 연결됨.
2. **일반 할 일에도 세부 업무**: **모든 할 일**에 대해 ▼를 눌러 펼치면, **같은 방식으로 세부 업무를 추가·관리**할 수 있음. "메인 업무 → 파생 세부 업무"를 한 곳에서 관리 가능.
3. **진행률**: 세부 업무가 하나라도 있으면 메인 카드에 **n/m 완료** 프로그레스 바로 표시.

**정리**: 메인 업무(목표 또는 일반 할 일) 옆 **▼ 펼치기** → 세부 업무 목록 확인 및 **"하위 업무 추가"**로 파생 업무 입력·관리.

### 2.4 피드백·로딩
- **문제**: 저장/삭제 후 "저장됐나?" 불안감, 로딩 시 빈 화면처럼 보임.
- **개선**: 토스트("추가됐어요" 등), 스켈레톤/로딩 문구, 실패 시 에러 메시지. (일부 이미 적용됨.)

### 2.5 캘린더·보기
- **문제**: 월간에서 날짜 셀 클릭 시 기대하는 동작(해당일 할 일 보기/이동)이 없음.
- **개선**: 날짜 셀 클릭 시 해당 날짜 할 일만 보여 주거나, 주간 뷰로 전환 후 그 주로 이동.

### 2.6 모바일
- **문제**: 필터·정렬이 많고, 추가 폼이 길어 세로 공간을 많이 씀.
- **개선**: 필터/정렬을 드롭다운 하나로 묶기, 추가는 기본 "제목 + [추가]"만 노출하고 마감·우선순위는 펼침으로.

## 3. 업무 효율성을 높이는 기능·방향

| 제안 | 설명 | 기대 효과 |
|------|------|-----------|
| **한 줄 빠른 추가** | 상단 고정: 제목만 입력 후 Enter/버튼 → 마감은 기본값 또는 추가 후 수정 | 할 일 입력 속도 향상, 진입 장벽 감소 |
| **일시 퀵 프리셋** | 오늘/내일/모레/금요일/다음주 월요일 등 버튼으로 한 번에 설정 | **(구현 완료)** 클릭 수 감소, 일시 입력이 간편해짐 |
| **메인–세부 한 화면** | 모든 할 일에서 ▼로 세부 업무 추가·진행률 확인 | **(구현 완료)** 업무 단위로 파악·실행 용이 |
| **검색·필터 강화** | 제목/설명 검색, "이번 주 마감만" 등 필터 저장(즐겨찾기) | 할 일이 많을 때 집중 대상 빠르게 찾기 |
| **반복 할 일** | 매일/매주/매월 반복 옵션 | 정기 업무(회의, 리뷰 등) 한 번 등록으로 관리 |
| **알림(푸시/이메일)** | 마감 N시간 전·당일 아침 등 알림 | 놓치기 쉬운 마감 감소 |
| **템플릿** | 자주 쓰는 할 일 묶음을 템플릿으로 저장 후 한 번에 추가 | 프로젝트·이벤트별 체크리스트 재사용 |
| **완료 시 간단 회고** | 완료 시 "소요 시간" "메모" 한 줄(선택) | 나중에 리포팅·개인 효율 분석에 활용 |
| **주간/일일 자동 요약** | "오늘 할 일 N건, 완료 M건" 등 대시 또는 이메일 | 업무량·완료율 가시화 |

## 4. 제도·시스템 제안 (앱을 활용한 업무 방식)

앱만 개선하는 것이 아니라, **사용 방식·제도**와 맞추면 업무 효율성을 더 끌어낼 수 있습니다.

### 4.1 개인 업무 습관
- **아침 5분**: 앱을 열어 "오늘 할 일"만 필터로 보고, 오늘 할 3~5건만 골라서 순서 정하기.
- **한 가지 끝나면**: 완료 체크 후, 다음 할 일을 목록에서 골라 진행. (목록/주간 뷰로 "다음"이 보이게.)
- **저녁 2분**: 미완료 건을 "내일로 미루기" 또는 마감일만 조정해 다음 날로 이어받기.

### 4.2 팀·조직에서의 활용
- **공유 목표**: 팀 공통 목표를 "목표"로 하나 두고, 각자 담당 세부 업무를 하위로 넣어 진행률을 한 화면에서 공유. (현재는 1인용이므로, 추후 "공유 보기" 또는 팀 스페이스가 있으면 적용.)
- **일일/주간 보고**: 매일 9시 이메일 보고서를 켜 두고, "어제 완료 · 오늘 예정 · 이번 주 마감"을 메일로 받아 팀 브리핑 자료로 사용.
- **마감일 규칙**: "목표는 분기 말, 세부 업무는 주 단위"처럼 마감일 규칙을 정해 두고, 앱에서는 퀵 프리셋·반복 할 일로 입력 부담을 줄이기.

### 4.3 정기 리뷰와 앱 개선
- **주 1회**: "이번 주 완료율", "미뤄진 건"을 앱에서 확인하고, 반복 할 일·템플릿을 정리.
- **월 1회**: "목표 대비 진행률"을 보고, 다음 달 목표·세부 업무를 다시 쪼개어 등록.
- **앱 개선 피드백**: 자주 쓰는 동작(예: "오늘 18:00 마감 많이 씀")은 퀵 프리셋·기본값에 반영해 입력 비용을 줄이기.

### 4.4 업무 효율을 위한 제도 예시
- **"3건 규칙"**: 매일 "반드시 끝낼 3건"만 상단에 두고 나머지는 선택. 앱에서는 우선순위·"오늘" 필터로 3건만 강조.
- **"목표–세부 분리"**: 큰 일은 반드시 목표 1개 + 세부 N개로 나누어 등록. 앱의 목표·하위 구조와 진행률을 그대로 활용.
- **"마감일 = 알림일"**: 마감일을 "이날까지 완료"로만 쓰지 않고, "이날 알림 보내기"로도 사용. (알림 기능 도입 시.)
- **보고서 수신 고정**: 매일 9시 보고서를 끄지 않고, 아침에 메일만 보고 "오늘 할 일"을 확인하는 습관화.

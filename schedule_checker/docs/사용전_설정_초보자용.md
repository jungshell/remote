# AutoFlow 사용 전 설정 — 초보자용 단계별 안내

앱을 처음 켜기 전에 **한 번만** 따라 하면 됩니다. 순서대로 하면 됩니다.

---

## 0. 준비물

- **Google 계정** (Gmail 로그인용)
- **프로젝트 폴더**가 열려 있는 상태 (예: `schedule_checker` 폴더)
- **Node.js**가 설치되어 있어야 함 (이미 `npm run dev`를 한 적 있다면 됨)

---

## 1단계: Firebase 프로젝트 열기

1. 웹 브라우저에서 **https://console.firebase.google.com** 접속
2. Google 계정으로 로그인
3. 왼쪽 상단이나 프로젝트 목록에서 **프로젝트 이름** 클릭  
   - 이 프로젝트 이름이 **schedule-checker-b0eb7** 이라면 그대로 선택  
   - 다른 이름의 프로젝트를 쓰려면 그 프로젝트를 선택하면 됨

> **Firebase가 뭔가요?**  
> 로그인(이메일/Google)과 데이터 저장(업무, 연락처 등)을 맡아 주는 서비스입니다. 우리 앱은 이걸 사용합니다.

---

## 2단계: 로그인 방법 켜기 (이메일 + Google)

1. Firebase 콘솔 왼쪽 메뉴에서 **「빌드」** → **「인증(Authentication)」** 클릭
2. **「시작하기」** 또는 **「로그인 방법」** 탭 클릭
3. **「이메일/비밀번호」** 줄 찾기  
   - **사용** 스위치를 **켜기**(파란색)  
   - 필요하면 **저장** 클릭
4. **「Google」** 줄 찾기  
   - **사용** 스위치를 **켜기**  
   - **프로젝트 지원 이메일** 선택 후 **저장**

이렇게 하면 **이메일로 가입/로그인**과 **Google 버튼으로 로그인**이 가능해집니다.

---

## 3단계: 웹 앱 설정값 복사하기 (가장 중요)

앱이 Firebase에 접속하려면 **설정값 6개**가 필요합니다. Firebase 콘솔에서 복사합니다.

1. Firebase 왼쪽 메뉴에서 **⚙️ 톱니바퀴** → **「프로젝트 설정」** 클릭
2. 아래로 내려서 **「내 앱」** 섹션 찾기
3. **웹 아이콘(</>)** 이 있는 **「웹 앱」**이 이미 있으면 그걸 클릭  
   - 없으면 **「앱 추가」** → **「웹(</>)」** 선택 → 앱 닉네임 아무거나 입력 후 **「앱 등록」**
4. **「SDK 설정 및 구성」** 또는 **「구성」** 탭 선택
5. **firebaseConfig** 안에 이런 값들이 보입니다. **그대로 복사**해 두세요.

   | 화면에 보이는 이름 | 복사할 변수 이름 (아래 4단계에서 쓸 이름) |
   |-------------------|-------------------------------------------|
   | apiKey            | NEXT_PUBLIC_FIREBASE_API_KEY              |
   | authDomain        | NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN          |
   | projectId         | NEXT_PUBLIC_FIREBASE_PROJECT_ID           |
   | storageBucket     | NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET       |
   | messagingSenderId | NEXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID  |
   | appId             | NEXT_PUBLIC_FIREBASE_APP_ID               |

   예시 (실제로는 본인 프로젝트 값이 나옵니다):

   ```js
   apiKey: "AIzaSy..."
   authDomain: "schedule-checker-b0eb7.firebaseapp.com"
   projectId: "schedule-checker-b0eb7"
   storageBucket: "schedule-checker-b0eb7.firebasestorage.app"
   messagingSenderId: "123456789"
   appId: "1:123456789:web:abc..."
   ```

이 6개 값을 **메모장이나 메모 앱에 붙여 넣어 두었다가** 다음 단계에서 사용합니다.

---

## 4단계: 프로젝트 폴더에 환경 변수 파일 만들기

앱이 **어디서 Firebase에 접속할지** 알려 주는 파일을 만듭니다.

1. **schedule_checker** 폴더를 연다 (Cursor에서 연 프로젝트 폴더와 같은 곳).
2. 그 **안에** 새 파일을 만든다.
   - 파일 이름: **`.env.local`** (맨 앞에 점(.)이 있고, 확장자는 .local)
   - Windows에서 점으로 시작하는 파일이 안 보이면: 메모장에서 내용 저장할 때 **파일 이름**을 ` .env.local` 로 하고, **파일 형식**을 **모든 파일(*.*)** 로 선택한 뒤 저장.
3. **.env.local** 파일을 메모장이나 Cursor로 열고, 아래 내용을 **복사**한 다음, **주석(#)과 빈 줄은 지우고**, 3단계에서 복사한 **실제 값**으로 바꿔 넣는다.

```env
NEXT_PUBLIC_FIREBASE_API_KEY=여기에_apiKey_값_붙여넣기
NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN=여기에_authDomain_값_붙여넣기
NEXT_PUBLIC_FIREBASE_PROJECT_ID=여기에_projectId_값_붙여넣기
NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET=여기에_storageBucket_값_붙여넣기
NEXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID=여기에_messagingSenderId_값_붙여넣기
NEXT_PUBLIC_FIREBASE_APP_ID=여기에_appId_값_붙여넣기
```

**주의**

- `=` 뒤에 **공백 없이** 값만 씁니다.
- 값에 **따옴표 넣지 않습니다.** (예: `AIzaSy...` 만 씀)
- 저장 후 **개발 서버를 한 번 종료했다가 다시 실행**해야 반영됩니다.

**예시 (실제 값은 본인 Firebase에서 복사한 것으로 바꾸세요):**

```env
NEXT_PUBLIC_FIREBASE_API_KEY=AIzaSyBxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN=schedule-checker-b0eb7.firebaseapp.com
NEXT_PUBLIC_FIREBASE_PROJECT_ID=schedule-checker-b0eb7
NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET=schedule-checker-b0eb7.firebasestorage.app
NEXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID=123456789012
NEXT_PUBLIC_FIREBASE_APP_ID=1:123456789012:web:abcdef123456
```

여기까지 하면 **로그인 + 업무 목록**을 쓸 수 있습니다.

---

## 5단계: Firestore 인덱스 한 번 배포하기 (목록이 안 나올 때)

업무·연락처 목록을 불러올 때 **“인덱스가 필요합니다”** 같은 오류가 나오면 이 단계를 합니다.

**방법 A — 터미널에서 한 번 실행 (권장)**

1. **schedule_checker** 폴더에서 터미널(또는 Cursor 터미널)을 연다.
2. 아래 명령을 입력하고 Enter 한다.

   ```bash
   firebase deploy --only firestore
   ```

3. 처음이면 **Firebase 로그인** 또는 **프로젝트 선택**을 요구할 수 있다. 안내대로 진행한다.
4. **배포가 완료**되면 브라우저에서 앱을 새로 고침한 뒤 다시 **업무** 목록을 연다.

**방법 B — 오류 메시지에 나온 링크로 만들기**

1. 앱을 실행한 뒤 **업무** 페이지(`/tasks`)로 간다.
2. 브라우저 **콘솔**(F12 → Console 탭)이나 화면에 **“인덱스를 만들어 주세요”** 같은 메시지와 **링크**가 뜬다.
3. 그 **링크를 클릭**하면 Firebase 콘솔의 “인덱스 만들기” 화면으로 간다.
4. **「인덱스 만들기」** 버튼을 눌러 완료될 때까지 기다린다(몇 분 걸릴 수 있음).
5. 완료되면 앱에서 목록을 다시 불러온다.

---

## 6단계: 앱 실행해서 로그인까지 확인

1. **AutoFlow ON.bat** 더블클릭 또는 터미널에서 `npm run dev` 실행
2. 브라우저에서 **http://localhost:4000** 접속
3. **「업무 관리로 이동」** 클릭 → 로그인 화면으로 이동
4. **이메일 + 비밀번호**로 가입/로그인 하거나 **「Google로 로그인」** 클릭
5. 로그인 후 **업무 목록**이 보이면 설정이 끝난 것입니다.

## 자주 묻는 것들

**Q. `.env.local`을 수정했는데 반영이 안 돼요.**  
→ 개발 서버를 **완전히 끄고**(Ctrl+C) 다시 `npm run dev` 또는 **AutoFlow ON.bat**으로 실행해 보세요.

**Q. “Firebase: Error (auth/configuration-not-found)” 같은 오류가 나요.**  
→ **3단계·4단계**를 다시 확인하세요. `NEXT_PUBLIC_FIREBASE_*` 값이 Firebase 콘솔의 **웹 앱 구성**과 **완전히 같고**, `.env.local`에 따옴표 없이 들어갔는지 봅니다.

**Q. 업무 목록이 안 나오고 “권한 없음” 또는 인덱스 오류가 나요.**  
→ **5단계**에서 `firebase deploy --only firestore`를 실행했는지, 또는 오류 메시지의 인덱스 링크로 인덱스를 만들었는지 확인하세요.

**Q. Google 캘린더 연동은 꼭 해야 하나요?**  
→ 아니요. **1~5단계만** 하면 로그인·업무·연락처·알림·템플릿은 쓸 수 있습니다. 캘린더는 나중에 **docs/Phase1-3_설정_요약.md**의 “Phase 2 – Google Calendar 연동”을 따라 하면 됩니다.

## 정리: 꼭 해야 하는 것만

| 순서 | 할 일 | 어디서 |
|------|--------|--------|
| 1 | Firebase 콘솔에서 **인증** → 이메일/비밀번호, Google **사용** | console.firebase.google.com |
| 2 | **프로젝트 설정** → 웹 앱 → **설정값 6개** 복사 | 같은 사이트 |
| 3 | 프로젝트 폴더에 **.env.local** 파일 만들고 6개 값 넣기 | schedule_checker 폴더 |
| 4 | (목록 오류 나면) **firebase deploy --only firestore** 실행 또는 오류 링크로 인덱스 생성 | 터미널 / Firebase 콘솔 |
| 5 | **AutoFlow ON.bat** 또는 **npm run dev**로 실행 → 로그인 테스트 | 브라우저 localhost:4000 |

이 문서는 **docs/사용전_설정_초보자용.md** 에 저장되어 있으니, 나중에 다시 보고 싶을 때 이 파일을 열면 됩니다.

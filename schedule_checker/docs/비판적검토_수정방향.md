# 비판적 검토 결과에 따른 수정 방향

비판적 검토를 바탕으로 **우선 적용할 수정 방향**을 정리했습니다. 이미 반영된 항목과 앞으로 할 작업을 구분해 두었습니다.

---

## 1. 이미 반영된 수정 사항

| 항목 | 내용 |
|------|------|
| **일시 입력** | 분은 **30분 단위**만 선택 가능. **오전 8시~오후 8시**만 노출, **오름차순**(오전 8시 → 오후 8시)으로 표시. |
| **기본 일시** | 할 일 추가 모달을 열었을 때 **현재 시각에 가장 가까운** 30분 단위 시각을 기본값으로 사용 (오전 1시 등 고정값 제거). |
| **퀵 프리셋** | 요청에 따라 퀵 프리셋 버튼 제거. |
| **날짜 + 요일** | 기한 날짜는 항상 **요일과 함께** 표시 (예: 3월 16일 (월)). |
| **담당자** | 할 일 추가 시 **담당자 선택** 필드 추가 (미지정, 나, 기존 할 일에서 쓴 담당자 목록). |
| **세부 업무 일시** | 메인 업무에 대한 **세부 업무**도 **마감 일시**를 지정할 수 있음. 세부 업무 목록에 마감일 표시. |
| **요일 한 줄** | 날짜 선택 시 요일이 **선택칸과 한 줄**로 표기 (두 줄 분리 제거). |
| **세부 업무 수정** | 파생(세부) 업무도 **제목·일시 수정** 가능. |
| **한 줄 빠른 추가** | 상단 **한 줄 입력 + [추가]**로 제목만 넣고 바로 추가. 상세는 "상세 옵션으로 할 일 추가". |
| **수정 모달** | 메인 할 일 수정 시 **모달**에서 편집 (인라인 제거). |
| **월간 → 주간** | 월간 뷰에서 날짜 셀 선택 후 **"주간 보기로 이동"** 버튼으로 해당 주 주간 뷰 전환. |
| **Google 캘린더** | 설정에서 **Google 캘린더 연동** 후, 할 일 추가 시 캘린더에 이벤트 자동 생성. |

## 2. 비판적 검토를 반영한 추가 수정 방향

### 2.1 진입·첫 화면
- **문제**: 보기/필터/대시가 먼저 보여 "할 일 하나 빨리 넣기"가 어렵고, 첫 화면이 복잡함.
- **수정 방향**:  
  - 상단에 **한 줄 입력 + [추가]**로 제목만 넣고 바로 추가 (마감은 기본값 또는 추가 후 수정).  
  - 보기/필터/대시는 **목록 아래**로 내리거나 **탭/접기**로 "기본은 단순, 필요할 때만 펼치기" 구조.

### 2.2 입력·편집 경험
- **문제**: 수정이 카드 안 인라인이라 카드가 길어지고, 수정/삭제 버튼이 작아 터치하기 어려움.
- **수정 방향**:  
  - 수정 시 **모달 또는 접이식 패널**에서 편집해 목록이 흐트러지지 않게 함.  
  - 수정/삭제는 **아이콘 버튼**으로 터치 영역 확보. 삭제는 **확인 모달** 유지.

### 2.3 피드백·로딩
- **문제**: 저장/삭제 후 "저장됐나?" 불안감, 로딩 시 빈 화면처럼 보임.
- **수정 방향**:  
  - 추가/수정/삭제 후 **토스트** 유지·강화.  
  - 목록 로딩 시 **스켈레톤** 또는 스피너로 "불러오는 중"이 분명히 보이게 함.  
  - 실패 시 **에러 메시지**를 토스트 또는 인라인으로 표시.

### 2.4 캘린더·보기
- **문제**: 월간 뷰에서 날짜 셀 클릭 시 아무 반응이 없어 기대에 맞지 않음.
- **수정 방향**:  
  - 날짜 셀 **클릭 시** 해당 날짜 할 일만 필터된 목록 표시 또는 **주간 뷰로 전환 + 그 주로 이동**.

### 2.5 모바일
- **문제**: 필터·정렬이 많고, 추가 폼이 길어 세로 공간을 많이 씀.
- **수정 방향**:  
  - 필터/정렬을 **드롭다운 하나**로 묶어 한 줄로 정리.  
  - 추가는 기본 **제목 + [추가]**만 노출하고, 마감·우선순위·담당자 등은 **펼침**으로 처리.

### 2.6 네비게이션
- **문제**: "업무"가 메인인지 한눈에 들어오지 않음.
- **수정 방향**:  
  - **"업무"를 메인**으로 두고, 연락처·알림·템플릿은 **서브 메뉴**(드롭다운·햄버거)로 묶어 기본 화면을 단순하게.

## 3. 우선 적용 순서 제안

| 순위 | 수정 방향 | 비고 |
|------|-----------|------|
| 1 | **한 줄 빠른 추가** (상단 고정, 제목만 입력 후 추가) | 진입 장벽 감소 |
| 2 | **로딩 스켈레톤·에러 메시지** 정리 | 체감 품질 상승 |
| 3 | **월간 셀 클릭 → 해당일 할 일 보기/주간 이동** | 캘린더 기대에 맞춤 |
| 4 | **수정을 모달/패널로** 전환, 수정/삭제 터치 영역 확대 | 편집 사용성 개선 |
| 5 | **필터/정렬 드롭다운화** (모바일) | 작은 화면에서 단정한 UI |
| 6 | **입력 패턴 기억 → 템플릿/기본값 제안** | `docs/입력패턴_템플릿화_제안.md` 참고 |

# AutoFlow 앱 업데이트 제안

현재 앱은 **업무 목록(제목·설명)** 중심이라, 스케줄 관리와 한눈에 파악하기에는 부족한 부분이 있습니다. 아래는 우선 적용해 볼 만한 개선 방향입니다.

---

## 1. 버그 수정 (적용 완료)

- **추가 시 오류**: `description`이 비어 있을 때 `undefined`가 Firestore에 전달되어 에러 발생 → **제목/설명을 빈 문자열로 저장하도록 수정함.** (이제 제목만 넣고 추가해도 동작합니다.)

---

## 2. 스케줄·일정 쪽 보강

| 제안 | 설명 |
|------|------|
| **마감일(dueDate)** | 할 일에 “언제까지” 필드 추가. 목록/캘린더에서 일정처럼 보이게. |
| **우선순위** | 상/중/하 또는 숫자. 목록 정렬·필터에 사용. |
| **캘린더 뷰** | 이미 연동된 Google 캘린더 + 할 일 마감일을 **주/월 뷰**로 보기. |
| **오늘 할 일 강조** | 마감일이 오늘인 항목을 상단 또는 별도 블록으로 표시. |

---

## 3. 한눈에 파악하기

| 제안 | 설명 |
|------|------|
| **대시 요약** | “오늘 할 일 n개”, “이번 주 마감 m개”, “완료율 %” 등 한 줄 요약. |
| **필터·정렬** | 완료/미완료, 마감일순, 우선순위순, 제목순 등. |
| **상태/라벨** | 진행 중·보류·완료 등 상태, 또는 태그로 구분해 한눈에 구분. |
| **목록 밀도** | 카드/리스트 전환, 중요 정보(마감일·우선순위)만 강조해 보기. |

---

## 4. 입력·편집 보강

| 제안 | 설명 |
|------|------|
| **마감일 선택** | 날짜(·시간) 선택 UI 추가. |
| **우선순위 선택** | 드롭다운/버튼으로 지정. |
| **설명 확장** | 긴 설명은 접기/펼치기, 또는 별도 모달. |
| **담당·태그(선택)** | 나중에 협업/분류 확장 시 사용. |

---

## 5. 연동·자동화

| 제안 | 설명 |
|------|------|
| **알림(Alerts) 연동** | 할 일 마감일과 연동해 “마감 D-day” 알림. |
| **템플릿 연동** | 자주 쓰는 업무를 템플릿으로 저장 후, 한 번에 할 일 여러 개 생성. |
| **일일 요약** | 이미 있는 `/api/automation/daily-summary`를 대시에 “어제 완료·오늘 예정”으로 노출. |

---

## 6. 우선순위 제안 (구현 순서)

1. **마감일(dueDate) 추가** + 목록에 표시 ✅
2. **오늘 할 일 / 대시 요약** ✅ (진행 중, 오늘 할 일, 이번 주 마감, 완료 수 표시)
3. **우선순위** + **필터·정렬** ✅ (높음/보통/낮음, 전체·진행 중·완료 필터, 마감일순/우선순위순/최신순 정렬)
4. **캘린더 뷰** (주/월) ✅ (목록 / 주간 / 월간 보기 전환)
5. **알림·템플릿 연동** ✅ (D-Day/D-1/마감 지남 뱃지, 연락처·알림·템플릿 링크)

원하시면 위 항목 중에서 “먼저 넣을 기능”을 정해 주시면, 그에 맞춰 구체적인 스펙(필드 설계, 화면)까지 정리해 드리겠습니다.

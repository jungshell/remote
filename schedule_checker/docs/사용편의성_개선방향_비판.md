# 사용자 이용 편의성 개선 방향 (비판적 정리)

지금 앱은 **기능은 많지만, 실제로 쓰기에는 불편한 점**이 많습니다. “할 일을 빨리 넣고, 확인하고, 수정하는” 흐름보다 **설정·보기·필터가 먼저 눈에 들어와** 진입 장벽이 높고, 피드백·오류 대응도 부족합니다. 아래는 **이용 편의성 우선**으로 비판적으로 정리한 개선 방향입니다.

---

## 1. 첫 화면·정보 구조 (진입 장벽)

### 문제
- 업무 페이지에 들어오면 **보기(목록/주간/월간) → 대시 4칸 → 추가 폼 → 필터/정렬 → 목록** 순서라, **“할 일 하나 빨리 넣고 싶다”**는 사용자에게는 클릭·스크롤이 많음.
- 할 일이 없을 때와 있을 때 레이아웃이 크게 바뀌어, **처음 쓰는 사람은 “뭘 먼저 해야 하지?”** 느낌을 받기 쉬움.
- **“새 할 일 추가”**가 화면 상단 고정이 아니라 폼이 중간에 있어, 목록이 길어지면 **스크롤 없이는 추가하기 어렵고**, “추가” 행동이 눈에 잘 안 들어옴.

### 개선 방향
- **메인 액션을 “할 일 추가”로 단순화**: 상단에 **한 줄 입력 + Enter 또는 버튼**으로 바로 추가할 수 있게 하고, 마감일·우선순위는 “추가 후 수정” 또는 접힌 “상세 옵션”으로 두기.
- **보기/필터/대시**는 **목록 아래**로 내리거나, **탭/접기**로 “기본은 단순, 필요할 때만 펼치기” 구조로 변경.
- **할 일이 0개일 때**: “첫 할 일을 입력해 보세요” 한 줄 입력만 보여 주고, 나머지 UI는 할 일이 생긴 뒤에만 노출.

## 2. 입력·편집 경험

### 문제
- **추가 후 성공/실패 피드백이 없음** → “저장됐나?” 불안감, 실패해도 이유를 모름.
- **수정이 카드 안 인라인**이라, 수정 모드일 때 카드가 길어지고 **저장/취소**가 작게만 보여 터치하기 어려움.
- **수정/삭제 버튼**이 텍스트만 있고 아이콘·영역이 작아 **모바일에서 누르기 힘들고**, 실수로 삭제할 위험도 있음.
- **삭제가 `confirm()` 한 번**이라, 잘못 눌렀을 때 복구 불가.

### 개선 방향
- **추가/수정/삭제 후** 짧은 **토스트 메시지** (“추가됐어요”, “수정됐어요”, “삭제됐어요”) 또는 인라인 메시지로 즉시 피드백.
- **수정**: 카드 클릭 시 **모달 또는 아래로 펼쳐지는 패널**에서 수정하게 하여, 목록이 흐트러지지 않고 “수정 영역”이 명확해지게 하기.
- **수정/삭제**: 카드 오른쪽에 **아이콘 버튼**(연필, 휴지통)으로 넓은 터치 영역 확보.
- **삭제**: 확인 **모달**에서 “정말 삭제할까요?” + “취소/삭제” 버튼으로 한 번 더 확인. (필요하면 “최근 삭제 복구”는 이후 단계에서.)

## 3. 로딩·저장 상태 (피드백 부족)

### 문제
- **목록 로딩** 시 “목록 불러오는 중...” 텍스트만 있어, **빈 화면처럼 보이거나** 로딩인지 오류인지 구분이 어려움.
- **저장/수정/삭제 중**에는 버튼만 `disabled`이고, **“저장 중…”, “삭제 중…”** 같은 문구가 없어 진행 여부를 알기 어려움.
- **네트워크·권한 오류** 시 사용자에게 보여 줄 메시지가 없거나 불명확할 수 있음.

### 개선 방향
- **목록 로딩**: **스켈레톤 카드** 또는 **스피너**를 써서 “지금 불러오는 중”임을 분명히 표시.
- **저장/수정/삭제**: 해당 버튼 옆 또는 카드 위에 **“저장 중…”, “삭제 중…”** 문구 노출.
- **실패 시**: “저장에 실패했어요. 다시 시도해 주세요.” 등 **짧은 에러 메시지**를 토스트나 인라인으로 표시.

---

## 4. 보기 모드·캘린더 (동작과 기대 불일치)

### 문제
- **월간 뷰**에서 날짜 셀을 눌러도 **아무 반응 없음** → “그날 할 일 보기/이동”을 기대하는 사용자에게 실망감.
- **주간 뷰**에서 할 일을 눌러도 **완료 토글만** 되고, **상세 보기·수정**으로 이어지지 않아 “클릭하면 뭔가 더 보이겠지” 기대에 안 맞음.
- **주간 7열**이 **모바일**에서 너무 좁아, 날짜와 제목이 잘 안 보일 수 있음.

### 개선 방향
- **월간**: 날짜 셀 **클릭 시** 해당 날짜 할 일만 필터된 목록으로 보여 주거나, **주간 뷰로 전환 + 그 주로 이동**.
- **주간**: 할 일 **클릭 시** 상세/수정 모달 또는 패널 열기; **완료 체크**는 체크박스만 따로 두어 역할 분리.
- **모바일**: 주간은 **가로 스크롤** 또는 **요일별 스와이프 카드**로 바꿔, 한 칸이 넓고 터치하기 쉽게.

---

## 5. 네비게이션·헤더

### 문제
- **홈 / 연락처 / 알림 / 템플릿**이 같은 급으로 나열돼 있어, **“업무(할 일)”가 메인**인지 한눈에 안 들어옴.
- **로그아웃**이 텍스트 링크뿐이라, 터치하기 쉽지 않고 “로그아웃인가, 다른 메뉴인가” 구분이 약함.

### 개선 방향
- **“업무”를 메인**으로 두고, 연락처·알림·템플릿은 **서브 메뉴**(드롭다운·햄버거 메뉴)로 묶어서 **기본 화면은 단순**하게.
- **로그아웃**은 버튼 형태로 두거나 아이콘+텍스트로, **클릭 영역을 넓게**.

---

## 6. 모바일·반응형

### 문제
- **필터·정렬 버튼**이 많아서 **한 줄에 안 들어가면** 여러 줄로 wrap되고, 작은 화면에서는 더 복잡해 보임.
- **추가 폼**에 입력칸이 많아 **모바일에서 세로 공간**을 많이 씀.
- **테이블형 레이아웃**(주간 7열, 월간 그리드)이 **작은 화면**에서 가독성이 떨어짐.

### 개선 방향
- **필터/정렬**은 **드롭다운 하나**로 묶기 (예: “전체 ▼”, “마감일순 ▼”) → 한 줄로 정리.
- **추가**: 기본은 **제목 한 줄 + [추가]**만 보이고, **“상세(마감일·우선순위)”**는 펼침/접기로 처리.
- **주간/월간**은 **모바일에서 레이아웃 전환**(스와이프·스크롤 강화) 또는 **간단한 목록형 대체 뷰** 제공.

---

## 7. 접근성·명확성

### 문제
- **“보기”, “필터”, “정렬”**이 라벨만 있고, **현재 선택값**이 무엇인지 처음 오는 사용자에게는 불명확할 수 있음.
- **수정/삭제**가 텍스트만 있어, **시각적으로 구분**이 약하고 **스크린 리더** 등에서도 불리할 수 있음.

### 개선 방향
- **선택된 보기/필터/정렬**을 **강조**(색·테두리·체크)하고, 필요하면 **“지금 보기: 진행 중만”** 같은 한 줄 안내 추가.
- **수정/삭제**에 **아이콘 + aria-label**을 넣어, 보는 사람과 보조 기기가 모두 “수정”, “삭제”로 인식하게 하기.

## 8. 우선 적용하면 좋은 순서 (이용 편의성 기준)

| 순위 | 개선 항목 | 기대 효과 |
|------|-----------|-----------|
| 1 | **한 줄 빠른 추가** (제목 + Enter/버튼, 상단 고정) | 진입 장벽 감소, “할 일 넣기”가 가장 쉬운 행동이 됨 |
| 2 | **추가/수정/삭제 후 토스트 피드백** | “저장됐다/안 됐다” 불안 해소 |
| 3 | **저장 중/로딩 중 문구·스켈레톤** | 진행 상태 인지, 빈 화면 오해 방지 |
| 4 | **수정/삭제를 아이콘 버튼 + 삭제 확인 모달** | 터치 용이, 실수 삭제 감소 |
| 5 | **첫 화면 단순화** (추가 강조, 보기/필터 하단 또는 접기) | “뭘 먼저 하지?” 해소 |
| 6 | **월간 셀 클릭 → 해당일 할 일 보기/주간 이동** | 캘린더 기대에 맞는 동작 |
| 7 | **필터/정렬 드롭다운화 (모바일 대응)** | 작은 화면에서도 단정한 한 줄 UI |
| 8 | **헤더에서 “업무” 강조, 연락처 등 서브 메뉴화** | 메인 기능이 분명해짐 |

---

## 요약

- **지금 앱**은 기능 수는 많지만, **“할 일 추가 → 확인 → 수정/삭제”**라는 **핵심 흐름이 눈에 잘 안 들어오고**, **피드백과 오류·로딩 표현이 부족**해 이용성이 떨어져 보입니다.
- **사용자 이용 편의성을 우선**하면, **진입 단계 단순화(한 줄 추가)**, **명확한 피드백(토스트·로딩·에러)**, **수정/삭제 사용성(아이콘·확인 모달)**, **보기/캘린더 동작 정리(클릭 의미 부여)** 순으로 손보는 것이 효과적입니다.

원하시면 위 순서대로 **구체적인 UI/코드 수정안**부터 적용해 드리겠습니다.

## 10. 모바일 환경 전용 개선 (완전 대응)

### 터치·입력
- **터치 타겟 최소 44×44px**: 모든 버튼·링크가 엄지로 누르기 쉬운 크기.
- **하단 안전 영역(Safe area)**: 노치·홈 인디케이터 영역 피해서 버튼·입력창 배치.
- **입력 시 스크롤**: 포커스된 입력창이 키보드에 가리지 않도록 자동 스크롤.
- **숫자/날짜 입력**: `input type="date"` 등이 모바일에서 네이티브 피커로 열리도록 유지.

### 레이아웃·보기
- **주간 뷰 모바일**: 7열 대신 **요일 가로 스크롤** 또는 **스와이프로 요일 전환** (한 요일씩 넓게).
- **월간 뷰 모바일**: 셀 크기·글자 크기 확대, 또는 **리스트형 "이번 달 할 일"**로 대체.
- **FAB(플로팅 액션 버튼)**: 오른쪽 하단 "할 일 추가" 버튼으로 어디서든 빠른 추가.
- **하단 탭/네비**: 업무 / 연락처 / 알림 등 메인 화면을 **하단 탭**으로 전환 (한 손 조작).

### 성능·체감
- **가상 스크롤**: 할 일이 100개 이상일 때 목록은 화면에 보이는 만큼만 렌더링해 끊김 방지.
- **터치 반응**: 버튼 터치 시 **즉시 시각 피드백**(opacity·scale)으로 눌렸다는 느낌.
- **제스처 피드백**: 스와이프 시 "삭제/완료" 아이콘이 따라오는 등 진행도 표시.

### PWA·설치
- **홈 화면 추가**: PWA 매니페스트·서비스 워커로 "앱처럼" 설치 가능.
- **스플래시·테마 컬러**: 설치 후 아이콘·스플래시로 앱 느낌 유지.

## 11. 감각적인 UI/UX (비주얼·톤앤매너)

### 시각적 톤
- **타이포 위계**: 제목·부제·본문·캡션 글자 크기·굵기 차이로 정보 우선순위가 보이게.
- **색채 체계**: 메인 포인트 컬러 1개 + 강조(위험/성공/경고) 색을 일관되게. 그라데이션·그림자는 절제.
- **여백·그리드**: 카드 간격·패딩을 8px 단위 등으로 통일해 정돈된 인상.
- **아이콘 통일**: 선형 또는 채우기 중 하나로 통일, 크기(예: 20–24px) 고정.

### 감각을 살리는 디테일
- **카드**: 살짝 둥근 모서리(12–16px), 은은한 그림자 또는 테두리로 "덩어리"가 보이게.
- **강조 요소**: CTA 버튼만 색+여백으로 돋보이게, 나머지는 중성 톤.
- **D-day·우선순위**: 색만으로 구분(빨강/노랑/회색) + 필요 시 작은 아이콘으로 직관성 보강.
- **다크 모드**: 배경·텍스트 대비 유지, 포인트 컬러는 채도 조절해 눈이 편하게.

### 일관성
- **전 페이지 공통**: 헤더 높이, 버튼 스타일, 카드 스타일, 폰트를 앱 전반에 동일 적용.
- **상태 표현**: 비활성/로딩/에러 상태마다 같은 패턴(회색 처리, 스피너, 빨간 문구 등).

## 12. 인터랙션으로 경험 높이기

### 마이크로 인터랙션
- **버튼 클릭/터치**: 눌렀을 때 `scale(0.98)` 또는 opacity 0.8 등 **즉각 반응**.
- **체크박스**: 체크 시 ✓ 그려지는 **짧은 애니메이션**(0.2s 내외).
- **토스트 등장/퇴장**: 아래에서 살짝 올라오거나 fade in/out (0.25s).
- **카드 호버/포커스**: 데스크톱 호버 시 살짝 들어올리거나 테두리 강조.

### 목록·전환
- **목록 등장**: 새로 불러온 항목이 **stagger**(0.05s 간격)로 차례로 나타나기.
- **추가 시**: 새 할 일 카드가 **한 장 슬라이드 인** 또는 fade in 되도록.
- **삭제 시**: 카드가 **슬라이드 아웃 + 높이 접히기** 후 목록 재정렬 (약 0.3s).
- **수정 모드 진입**: 인라인 수정 시 해당 카드만 **살짝 펼쳐지며** 포커스.

### 페이지·모달
- **모달 열기/닫기**: 배경 dim + 모달 scale/fade. 닫을 때 역방향.
- **페이지 전환**: Next.js 라우트 전환 시 **fade 또는 짧은 슬라이드**로 화면이 바뀌는 느낌.
- **스크롤 위치**: 목록에서 상세로 갔다가 뒤로 오면 **이전 스크롤 위치 복원**.

### 제스처(모바일)
- **스와이프로 삭제/완료**: 카드 좌측 스와이프 시 "삭제" 빨간 영역, 우측 스와이프 시 "완료" 초록 영역 노출 후 액션.
- **끌어당겨 새로고침**: 목록 최상단에서 당기면 로딩 인디케이터 + 새로고침.
- **길게 누르기(Long press)**: 카드 길게 누르면 컨텍스트 메뉴(수정/삭제/복제) 등.

### 로딩·대기
- **스켈레톤**: 카드 형태로 **펄스 애니메이션** 적용해 "곧 채워질 자리"가 보이게.
- **버튼 로딩**: 제출 중 버튼에 **스피너 + "저장 중…"**으로 중복 클릭 방지·상태 전달.
- **무한 스크롤**: 아래 도달 시 로딩 인디케이터로 "더 불러오는 중" 표현.

| 구분 | 항목 | 비고 |

| **필수(이용성)** | 1~8번 (기존) | 진입·피드백·수정/삭제·캘린더·네비·모바일 기본 |
| **추가 기능** | 9~19 (다크 모드 제외) | 빈 상태, 새로고침, 검색, 순서 변경, 아코디언, 제스처, 전환 등 |

| **인터랙션** | 12절 전체 | 버튼/체크 반응, 목록 등장·삭제 애니메이션, 모달 전환, 스와이프, 스켈레톤 |

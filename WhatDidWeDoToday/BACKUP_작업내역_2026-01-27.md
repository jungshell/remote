## 주요 변경 사항

### 1. 4컷 만화 이미지 자동 생성

#### 캐릭터 일관성 강화
- **파일**: `app/src/app/api/diary/route.ts`, `app/src/app/api/diary/regenerate-4cut/route.ts`
- **변경**: `FIXED_CHARACTER_DESCRIPTIONS` 프롬프트 강화
  - "옷만 다를 뿐, 4컷 모두·다른 날짜에서도 동일한 캐릭터로 그린다"
  - "같은 얼굴·같은 머리·같은 체형으로 같은 인물임이 한눈에 보이게"
  - 아빠: 수염 필수, 4컷 모두 동일한 한 사람
  - 엄마/아이: 4컷 모두 동일한 캐릭터
  - 의사 등 다른 성인 남성 등장 가능하나, 가족 아빠는 수염 있는 한 명만

#### 이미지 생성 서비스
- **나노바나나만 사용**: Pollinations, HuggingFace 이미지 생성 코드 모두 제거
- **파일**: 
  - `app/src/app/api/diary/route.ts` - Pollinations/HuggingFace 제거
  - `app/src/app/api/diary/regenerate-4cut/route.ts` - Pollinations 제거
- **동작**: `NANOBANANA_API_KEY`가 있으면 나노바나나만 사용, 없으면 이미지 생성 안 함

#### 이미지 Storage 저장
- **파일**: `app/src/lib/saveDiaryImage.ts` (신규)
- **기능**: 나노바나나에서 받은 이미지 URL을 Firebase Storage에 저장해 영구 보관
- **경로**: `diaries/{일기id}/4cut.png` (또는 webp/jpg)
- **적용**: 4컷 생성 시 자동으로 Storage에 저장 후, Storage URL을 `combinedImageUrl`에 저장

#### 재생성 시 기존 이미지 보호
- **파일**: `app/src/app/api/diary/regenerate-4cut/route.ts`
- **변경**: 나노바나나 실패 시 기존 `combinedImageUrl` 유지 (빈 값으로 덮어쓰지 않음)
- **문제 해결**: 재생성 시 나노바나나 실패해도 기존 이미지가 사라지지 않도록 수정

### 2. 한글 캡션 오탈자 개선

#### 캡션 생성 프롬프트 강화
- **파일**: `app/src/lib/llm.ts` - `generateImageCaptions()`
- **추가된 내용**:
  - 흔한 오기 예시 명시: 독서(O)/독조(X), 출발(O)/즉랄(X), 꽈배기(O)/꽈베기(X) 등
  - 국립국어원 표준어만 사용 지시

#### 자동 오기 수정 테이블
- **파일**: `app/src/lib/llm.ts` - `correctKoreanCaptionTypos()`
- **기능**: 생성된 캡션에서 흔한 오기를 자동으로 표준어로 치환
- **수정 항목**: 꽈베기→꽈배기, 독조→독서, 즉랄→출발, 차품 타고→차를 타고, 주솨→주사, 맛기→맞기, 식솨→식사, 엄빠와→엄마와 아빠와, 맴마→엄마, 등원 주께→등원 준비, 쓱쓱하케모→씩씩하게, 만족들론은→만족스러운, 전력→저녁 등

#### LLM 2단계 맞춤법 보정
- **파일**: `app/src/lib/llm.ts` - `correctCaptionsSpellingWithLLM()`
- **기능**: 캡션 생성 후 LLM으로 한 번 더 맞춤법·띄어쓰기만 교정
- **적용**: `generateImageCaptions()` 내부에서 자동 호출

#### 적용 위치
- **새 캡션 생성 시**: `generateImageCaptions()` → 고정 테이블 → LLM 보정 → 고정 테이블
- **기존 일기 재생성 시**: `regenerate-4cut` API에서도 동일한 보정 적용

## API 엔드포인트

### 4컷 재생성
- **POST** `/api/diary/regenerate-4cut`
- **Body 옵션**:
  - `{ "id": "일기id" }` - 특정 일기만
  - `{ "date": "2026-01-26" }` - 날짜로 찾기
  - `{ "all": true }` - 전체 재생성
  - `{ "allExceptDate": "2026-01-26" }` - 특정 날짜 제외 전체 재생성
  - `{ "saveExistingImages": true }` - 기존 외부 URL을 Storage에 저장

### 일기 저장/수정
- **POST** `/api/diary` - 새 일기 생성
- **PATCH** `/api/diary` - 일기 수정

필수:
- `FIREBASE_SERVICE_ACCOUNT` - Firebase Admin JSON

## 알려진 이슈 및 해결 방법

### 1. 이미지가 안 나오는 경우
- **원인**: 나노바나나 API 크레딧 부족 (402)
- **로그 확인**: 서버 터미널에서 `[Nanobanana]` 검색
- **해결**: 나노바나나 계정 크레딧 충전 후 재생성

### 2. 재생성 시 기존 이미지가 사라지는 경우
- **원인**: 나노바나나 실패 시 빈 값으로 덮어쓰기 (이미 수정됨)
- **현재**: 나노바나나 실패 시 기존 URL 유지

### 3. Firebase Storage 버킷 없음
- **로그**: "The specified bucket does not exist" (404)
- **해결**: Firebase Console에서 Storage 버킷 생성 및 `FIREBASE_STORAGE_BUCKET` 설정

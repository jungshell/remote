# 무료 플랜으로 계속 사용하기 가이드

## 현재 상황
- **읽기**: 일일 50,000회 사용 (100% 초과)
- **할당량 리셋**: 매일 자정 UTC (한국 시간 오전 9시)

## 할당량 리셋 시간

### 다음에 앱을 확인할 수 있는 때
- **매일 자정 UTC** (한국 시간 **오전 9시**)에 할당량이 리셋됩니다
- 예: 오늘 1월 28일 오후 12시에 초과했다면 → **내일 1월 29일 오전 9시**에 리셋

### 할당량 리셋 확인 방법
1. Firebase 콘솔 → 사용량 및 결제
2. 날짜 선택기에서 오늘 날짜 확인
3. 자정(UTC) 이후에 사용량이 0으로 리셋되는지 확인

## 무료로 계속 사용하기 위한 최적화

### 1. 클라이언트 캐싱 (적용 완료)
- 일기 목록을 localStorage에 10분간 캐싱
- 같은 페이지에서 10분 이내 재방문 시 API 호출 없음
- **효과**: 약 50-70% 읽기 감소

### 2. API 호출 최적화 (적용 완료)
- 일기 목록 limit: 50개 → 30개
- motivation API: 100개 → 30개, 1분 → 5분
- 불필요한 API 호출 제거

### 3. 사용자 행동 권장사항

#### ✅ 권장 사항
- 같은 세션에서 여러 번 새로고침하지 않기
- 일기 목록을 본 후 10분 이내에는 캐시 사용
- 탭 전환을 최소화 (각 탭마다 API 호출)

#### ❌ 피해야 할 행동
- 페이지를 계속 새로고침하기
- 여러 탭에서 동시에 열기
- 개발 중 반복적인 테스트 (할당량 소모)

### 4. 할당량 모니터링
- Firebase 콘솔에서 "사용량 및 결제" 확인
- 일일 사용량이 30,000회 이상이면 주의
- 40,000회 이상이면 추가 최적화 필요

## 예상 사용량 (최적화 후)

### 일반적인 사용 시나리오
- **페이지 로드 1회**: 약 30-60회 읽기
  - 일기 목록: 30회
  - motivation: 30회
- **일기 상세보기**: 1회 읽기 (캐시 사용 시 0회)
- **일기 작성**: 1회 쓰기

### 일일 예상 사용량
- **일기 10개 확인**: 약 300-600회 읽기
- **일기 5개 작성**: 약 5회 쓰기
- **총 예상**: 약 500-1,000회 읽기/일

**무료 할당량 50,000회 대비 약 1-2% 사용** → 충분히 여유있음!

## 할당량 초과 시 대응

### 즉시 조치
1. 페이지를 닫고 몇 시간 대기
2. 할당량 리셋 시간 확인 (오전 9시)
3. 다음 날 오전 9시 이후 접속

### 장기 대응
- 캐싱 시간을 더 늘리기 (10분 → 30분)
- limit을 더 줄이기 (30개 → 20개)
- motivation API 호출을 더 줄이기 (5분 → 10분)

## 추가 최적화 가능 사항

1. **서버 사이드 캐싱**: Vercel Edge Cache 활용
2. **필드 선택**: 필요한 필드만 읽기 (현재는 전체 문서)
3. **배치 읽기**: 여러 일기를 한 번에 읽기
4. **오프라인 지원**: Service Worker로 오프라인 캐싱

# Firestore 할당량 초과 원인 분석

## 현재 상황
- 읽기: 308회 (일일 할당량 50,000회의 0.6%)
- 할당량 초과 발생

## 할당량 초과 가능한 원인

### 1. 다른 할당량 초과 가능성
Firestore 무료 플랜의 제한:
- **읽기**: 일일 50,000회
- **쓰기**: 일일 20,000회
- **삭제**: 일일 20,000회
- **저장 용량**: 1GB
- **동시 연결**: 100개

**읽기 308회로는 할당량 초과가 발생하지 않습니다.** 따라서 다음 가능성이 있습니다:
1. **쓰기 할당량 초과**: 43회 쓰기가 있었지만, 실제로는 더 많을 수 있음
2. **저장 용량 초과**: 문서 크기가 너무 커서 1GB 제한 초과
3. **동시 연결 초과**: 100개 동시 연결 제한 초과
4. **다른 리소스**: Storage, Functions 등의 할당량 초과

### 2. 코드에서 발견된 비효율성

#### 문제 1: 재시도 로직이 읽기를 증가시킴
- 할당량 초과 시 3회 재시도 → 각 재시도마다 쿼리 실행 → 읽기 3배 증가
- **해결**: 재시도 로직 제거 (할당량 초과 시 즉시 에러 반환)

#### 문제 2: 인덱스 오류 시 여러 번 시도
- createdAt 인덱스 실패 → date 인덱스 시도 → 정렬 없이 시도
- 최대 3번의 쿼리 실행 가능
- **해결**: 이미 적용됨 (한 번만 시도)

#### 문제 3: motivation API가 너무 자주 호출
- 페이지 로드 시마다 호출
- 1분마다 자동 갱신
- 최근 100개 일기 읽기
- **해결**: 
  - 100개 → 30개로 감소
  - 1분 → 5분으로 변경
  - 일기 목록이 없으면 호출하지 않음

#### 문제 4: ReadFullScreen에서 불필요한 API 호출
- 일기 상세보기마다 API 호출
- prop에 이미 데이터가 있어도 호출
- **해결**: 필요한 경우에만 호출 (이미지가 없을 때만)

#### 문제 5: 여러 API 동시 호출
- 페이지 로드 시: `/api/diary`, `/api/motivation` 동시 호출
- 탭 전환 시: `/api/stats`, `/api/timeline` 등 추가 호출

## 최적화 적용 사항

1. ✅ 재시도 로직 제거 (할당량 초과 시 즉시 에러 반환)
2. ✅ motivation API 호출 빈도 감소 (1분 → 5분)
3. ✅ motivation API 읽기 수 감소 (100개 → 30개)
4. ✅ ReadFullScreen 불필요한 API 호출 방지
5. ✅ 일기 목록이 없으면 motivation API 호출 안 함

## 예상 효과

- **motivation API**: 100회 → 30회 읽기 (70% 감소)
- **재시도 제거**: 할당량 초과 시 읽기 증가 방지
- **불필요한 호출 제거**: 약 20-30% 추가 절약

## 추가 권장 사항

1. **캐싱 추가**: 클라이언트에서 일기 목록 캐싱 (5분)
2. **배치 읽기**: 여러 일기를 한 번에 읽기
3. **필드 선택**: 필요한 필드만 읽기 (현재는 전체 문서 읽기)
4. **Firebase 콘솔 확인**: 실제 할당량 사용량 확인

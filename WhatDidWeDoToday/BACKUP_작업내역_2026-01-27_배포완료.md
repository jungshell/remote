## 🔧 주요 변경 사항

### 1. 타임아웃 문제 해결 (클라이언트 사이드 폴링)

#### 문제점
- Vercel 무료 플랜: 10초 타임아웃
- 나노바나나 API 폴링: 최대 2분 소요
- 서버에서 폴링 시 타임아웃 발생

#### 해결 방법
클라이언트 사이드 폴링으로 전환:
- 서버는 taskId만 즉시 반환 (10초 내 완료)
- 브라우저에서 폴링 (타임아웃 없음)
- 완료 후 Firebase Storage에 저장

#### 새로 추가된 파일
1. **`app/src/lib/nanobananaClient.ts`**
   - 클라이언트 사이드 폴링 유틸리티
   - `startNanobananaTask()` - 나노바나나 시작
   - `pollNanobananaTask()` - 폴링 (최대 2분)

2. **`app/src/app/api/nanobanana/start/route.ts`**
   - 나노바나나 이미지 생성 시작 API
   - taskId만 즉시 반환

3. **`app/src/app/api/nanobanana/poll/route.ts`**
   - taskId 폴링 API (한 번만 체크)
   - 각 호출은 10초 내 완료

4. **`app/src/app/api/diary/save-image/route.ts`**
   - 생성된 이미지를 Firebase Storage에 저장

#### 수정된 파일
1. **`app/src/app/api/diary/route.ts`**
   - 나노바나나 폴링 제거
   - `combinedImagePrompt`만 생성하고 즉시 반환
   - 클라이언트에서 폴링 시작

2. **`app/src/app/api/diary/regenerate-4cut/route.ts`**
   - 나노바나나 폴링 제거
   - `combinedImagePrompt`만 생성하고 즉시 반환
   - `saveCombinedImageToStorage` import 추가

3. **`app/src/app/page.tsx`**
   - 일기 저장 후 클라이언트에서 나노바나나 폴링 시작
   - 백그라운드에서 폴링 → 완료 후 PATCH로 업데이트

4. **`app/src/lib/nanobanana.ts`**
   - `generateImageWithNanobanana` 함수에 deprecated 주석 추가
   - 서버에서 폴링하지 않도록 변경

### 2. 빌드 오류 수정

#### 오류 1: 변수명 중복
- **파일**: `app/src/app/api/nanobanana/poll/route.ts`
- **문제**: `url` 변수가 중복 정의됨
- **해결**: 
  - 14번 줄: `url` → `requestUrl`
  - 62번 줄: `url` → `imageUrl`

#### 오류 2: Import 누락
- **파일**: `app/src/app/api/diary/regenerate-4cut/route.ts`
- **문제**: `saveCombinedImageToStorage` 함수 import 누락
- **해결**: `import { saveCombinedImageToStorage } from "@/lib/saveDiaryImage";` 추가

- **문제**: `FIREBASE_SERVICE_ACCOUNT env is missing`
